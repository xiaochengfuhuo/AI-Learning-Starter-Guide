# 人工智能入门学习指南

> 本指南根据《入门文档》整理并补充，适合刚开始学习 Python 和人工智能的同学。核心原则是：**先补齐 Python 基础，再沿主线系统学习；每一课都要完成“看、读、跑、练、测、复盘”闭环。**

## 一、总体目标

| 阶段        | 主要任务                                         | 完成标志                                   |
| ----------- | ------------------------------------------------ | ------------------------------------------ |
| Python 基础 | 用约 7 天掌握基础语法                            | 能独立完成小练习，并看懂基础 Python 代码   |
| AI 公共基础 | 完成配置、数学、机器学习和深度学习核心内容       | 能解释主要概念，运行并修改课程代码         |
| 方向学习    | 根据目标学习 Transformer、生成式 AI 或计算机视觉 | 完成对应课程练习，留下代码、笔记和阶段总结 |

### 2. 优先学习路线

原文路线图中红框标出的优先阶段为：

1. `00 配置与工具链`
2. `01 数学基础`
3. `02 机器学习基础`


- 如果暂时没有明确方向，可以先走 `07 → 08`，再补 `04`，也可以按老师要求完成所有红框阶段。

## 四、每一课的标准学习流程

### 第 1 步：看清课程要求

开始前先确认：

- 前置课程是否已经掌握；
- 本课预计用时和学习目标；
- 本课属于 `Learn` 还是 `Build`；
- 需要运行哪些代码、提交哪些练习和产出。

预计用时只用于排期。初学者超过预计时间很正常，关键是确认时间花在理解、调试和练习上，而不是停留在被动观看。

### 第 2 步：视频与文档结合学习

1. 先快速浏览标题、目标和问题背景。
2. 观看讲解视频，记录不理解的关键词和公式。
3. 阅读课程文档，补齐视频中略过的推导和细节。
4. 用自己的话写出本课的核心结论，避免直接复制原文。

### 第 3 步：真正运行代码

- 找到该课程目录中的 `code/` 文件。
- 先原样运行，确认环境和依赖正常。
- 逐段阅读，给关键变量、函数和数据流加注释。
- 修改一个参数或实现，观察结果如何变化。
- 遇到报错时，保留报错信息和解决过程，形成调试笔记。

### 第 4 步：完成练习和测验

学习完成后，必须完成课程目录中的练习、课后测验和要求的产出。不要只看答案：先独立思考并写出自己的实现，再对照资料检查。

### 第 5 步：检查本课产出

课程常见目录结构如下：

```text
phases/<阶段>/<课程>/
├── code/       # 可运行代码
├── docs/       # 课程文档
└── outputs/    # 本课产出的提示词、skill、agent 等
```

完成后至少保留：

- 一份可运行的代码；
- 一页学习笔记；
- 练习或测验结果；
- 一个“仍不理解的问题”清单；
- 一段 3～5 句的课程总结。

### 第 6 步：课后复盘

每课结束后回答以下问题：

1. 我今天真正学会了什么？
2. 哪个概念我能不看资料讲清楚？
3. 哪段代码我能独立重写？
4. 练习是否全部完成？错误原因是什么？
5. 还有什么没有理解，下一步如何解决？

## 五、如何正确使用大模型辅助学习

看不懂时可以询问大模型，但目标是获得解释、提示和检查，而不是直接代写答案。可使用下面的提问模板：

```text
我正在学习【课程/主题】。

我目前理解的是：【用自己的话描述】
我不理解的是：【指出公式、概念、代码或报错】
相关代码/报错：【粘贴最小必要片段】

请你：
1. 先用直观例子解释；
2. 再逐步解释原理；
3. 给一个最小可运行示例；
4. 不直接给课后题最终答案，只给提示；
5. 最后用 3 个问题检查我是否真正理解。
```

使用大模型后，必须自己重新表述结论，并独立运行或重写代码。涉及公式、API 和运行结果时，应以课程材料、官方文档和实际执行结果为准。

## 七、阶段性验收标准

完成一个阶段后，不要只以“视频看完”为标准。至少满足以下条件：

- [ ] 能用自己的话解释本阶段的核心概念。
- [ ] 能运行所有重点代码，并说明输入、处理过程和输出。
- [ ] 能独立重写至少一个关键实现。
- [ ] 课程练习和测验已经完成。
- [ ] 对错误和调试过程有记录。
- [ ] 整理出一份阶段总结和待复习清单。
- [ ] 知道下一阶段为什么需要这些前置知识。

建议每天学习结束后用 10 分钟复盘，每周再用 30～60 分钟整理错题、代码和未解决问题。
